APP-BLE

User Guide

BLE Configuration

The default configuration of BLE in the project is Off.

Users can change configurations in APP_CompileOption.h to enable BLE.

Required Configuration

Value

APP_FREERTOS_ENABLE

APP_FALSE: Off

APP_TRUE: On

APP_BLE_ENABLE

APP_FALSE: Off

APP_TRUE: On

BLE Application Configurations

· Enable BLE

BLE application codes are localized in the AppBLE.c file, by default, the project is configured for peripheral application.

Users can configure which application to enable in AppBLE.h file.

Application Configuration

Value

BLE_HOST_ADVERTISER_APP_ENABLED

APP_FALSE: Off

APP_TRUE: On

BLE_HOST_PERIPHERAL_APP_ENABLED

APP_FALSE: Off

APP_TRUE: On

BLE_HOST_CENTRAL_APP_ENABLED

APP_FALSE: Off

APP_TRUE: On

BLE_HOST_SCANNER_APP_ENABLED

APP_FALSE: Off

APP_TRUE: On

In addition to configuring which application to use, users can also modify different parameters in the codes to implement their various demands.

Following are examples of how to modify the parameters in the AppBLE.c file to adjust the application’s advertisement function.

· Change Advertisement Parameters for Function: ble_gap_adv_start

Parameter

Meaning

own_addr_type

The type of address the stack should use.


Valid values:


BLE_OWN_ADDR_PUBLIC

BLE_OWN_ADDR_RANDOM

BLE_OWN_ADDR_RPA_PUBLIC_DEFAULT

BLE_OWN_ADDR_RPA_RANDOM_DEFAULT

direct_addr

The peer’s address for directed advertising.


Using directed advertising: non-NULL.

duration_ms

The duration of the advertisement procedure.


Unit: milliseconds.

Specify BLE_HS_FOREVER for no expiration.

adv_params

Additional arguments specifying the particulars of the advertising procedure:


conn_mode: Advertising mode.

disc_mode: Discoverable mode.

itvl_min: Minimum advertising interval.

itvl_max: Maximum advertising interval.

channel_map: Advertising channel map.

filter_policy: Advertising filter policy.

high_duty_cycle: High Duty cycle for Directed Advertising.

cb

The callback to associate with this advertising procedure.

cb_arg

The optional argument to pass to the callback function.

· Configure scanning parameters.

The behavior of the scan can be controlled through the structure ble_gap_disc_params.

Struct Field

Meaning

itvl

Scan interval in 0.625ms units.

window

Scan window in 0.625ms units.

filter_policy

Scan filter policy.

limited

If limited discovery procedure should be used.

passive

If passive scan should be used.

filter_duplicates

If enable duplicates filtering.

· Configure the name of the connected peripheral.

By default, only peripheral devices with the name “Test” can be connected, users can configure different device name, device UUID and other conditions to filter the connected peripheral device according to their own needs.

User can configure this in the central_scan_event function in AppBLE.c.

Examples

APP_BLE_Advertiser - Example

· Advertiser can be scanned when advertising.

APP_BLE_Peripheral - Example

· Peripheral can be scanned when advertising and can be connected.

APP_BLE_Central - Example

· Central can scan advertising devices and connect to peripheral.

APP_BLE_Scanner - Example

· Scanner can scan advertising devices.

· To see the scanner’s output, user needs to enable the APP_BLE_UARTPRINT_ENABLE configuration.

· In AppBLE.c, APP_BLE_UARTPRINT_ENABLE default to disabled, change from APP_FALSE to APP_TRUE to enable it.

· For the UART configuration, it is also available in the AppBLE.c file, there is a relevant content in the APP_BLE_UartInit() initialization function, please modify it according to your needs.
