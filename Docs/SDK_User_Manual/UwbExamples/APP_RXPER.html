APP-RXPER

概述

本文档提供了UWB TRX RX PER（Packet Error Rate，数据包错误率）测量的实现指南。通过该示例，用户可以测量和评估UWB接收器的性能，包括成功接收的数据包数量、错误包数量以及各种接收状态统计。

要求

  • 需要以下开发板中的一个：

    • CB5000V210 Anchor开发板

  • 软件：

    • 串行终端工具（例如PuTTY、SSCOM）

用户界面

  • 在每个开发板上找到GPIO00引脚，并将其连接到串行接收线（RX）。

  • 在每个开发板上找到GPIO01引脚，并将其连接到串行发送线（TX）。

  • 确保连接牢固，以避免通信问题。

构建和运行

  • 该示例可以在ChipsBank Connect SDK文件夹结构中的”Firmware\Examples\uwb_rxper”找到。

测试

  1. 连接开发板

    • 准备合适的USB线，确保两端与您的计算机和开发板兼容。

    • 将USB线的一端插入计算机的USB端口，另一端插入开发板的USB端口。

    • 打开计算机上的设备管理器，检查是否识别到新的COM端口。如果没有，请检查USB线连接并确保安装了正确的驱动程序。

  2. 配置串行终端

    • 在计算机上打开一个串行终端工具，例如PuTTY、SSCOM或类似工具。

    • 在每个串行终端工具中选择识别到的COM端口。

    • 将波特率设置为115200，停止位设置为1，数据位顺序设置为LSB，奇偶校验设置为NONE。

    • 点击连接按钮以建立与每个开发板的串行通信。

  3. 重置开发板

    • 在开发板上找到复位按钮。

    • 按下并释放复位按钮以重启开发板。

  4. 执行测试操作

    • 重置开发板后，在串行终端工具中观察输出。

    • 将开发板刷入uwb_rxper固件

    • 重置开发板后，设备将自动开始运行：

      • 在串行终端中显示”APP RX PER”

      • 显示UWB RX PER - num. of packets received in 10000 ms

      • 显示详细的包统计信息：

        • 总接收包数

        • 成功接收包数

        • 未接收包数

        • 各种接收状态统计（如rx0_ok, rx1_ok, rx2_ok等）

Overview

This document provides a guide to the implementation of UWB TRX RX PER (Packet Error Rate) measurement. Through this example, users can measure and evaluate the performance of the UWB receiver, including the number of successfully received packets, error packets, and various reception status statistics.

Requirements

  • One of the following development boards:

    • CB5000V210 Anchor demo board

  • Software:

    • Serial terminal tool (e.g., PuTTY, SSCOM)

User interface

  • Locate the GPIO00 pin on the development board and connect it to the serial receive line (RX).

  • Locate the GPIO01 pin on the development board and connect it to the serial transmit line (TX).

  • Ensure the connections are secure to avoid communication issues.

Building and running

  • This sample can be found under “Firmware\Examples\uwb_rxper” in the ChipsBank Connect SDK folder structure.

Testing

  1. Connect the Development Board

    • Prepare a suitable USB cable, ensuring both ends are compatible with your computer and demo board.

    • Insert one end of the USB cable into a USB port on your computer and the other end into the USB port on the demo board.

    • Open the Device Manager on your computer to check if a new COM port has been recognized. If not, verify the USB cable connection and ensure the correct drivers are installed.

  2. Configure the Serial Terminal

    • Open a serial terminal tool on your computer, such as PuTTY, SSCOM, or any similar tool.

    • In the serial terminal tool, select the recognized COM port.

    • Set the baud rate to 115200, stop bits to 1, data bits order to LSB, and parity to NONE.

    • Click the connect button to establish serial communication with the development board.

  3. Reset the Development Board

    • Locate the reset button on the development board.

    • Press and release the reset button to restart the development board.

  4. Execute Test Operations

    • After resetting the board, observe the output in the serial terminal tool.

    • Flash the development board with uwb_rxper firmware.

    • After resetting the board, the device will automatically start running:

      • Display “APP RX PER” in the serial terminal

      • Show “UWB RX PER - num. of packets received in 10000 ms”

      • Display detailed packet statistics:

        • Total received packets

        • Successfully received packets

        • Packets not received

        • Various reception status statistics (e.g., rx0_ok, rx1_ok, rx2_ok, etc.)
