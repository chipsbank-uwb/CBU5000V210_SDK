APP_ADC

概述

本文档提供了使用开发板（例如CB5000V210 Anchor开发板）验证ADC（模数转换器）外设功能的指南。它涵盖了硬件连接、串行通信设置以及ADC采样和电压转换的内容。

要求

  • 以下开发板之一：

    • CB5000V210 Anchor开发板

  • 软件：

    • 串行终端工具（例如PuTTY, SSCOM）

用户界面

  • 串口连接:

    • 将GPIO00连接到串行接收线（RX）。

    • 将GPIO01连接到串行发送线（TX）。

  • ADC输入连接:

    • 将开发板上的AIN（模拟输入）连接到您要测量的电压源（0-3.3V）。

  • 确保所有连接牢固，以避免通信问题。

构建和运行

  • 该示例可以在ChipsBank Connect SDK文件夹结构中的”Firmware\Examples\peripheral_adc”找到。

测试

  1. 连接开发板

    • 准备一个合适的USB线，确保两端与您的计算机和开发板兼容。

    • 将USB线的一端插入计算机的USB端口，另一端插入开发板的USB端口。

    • 打开计算机上的设备管理器，检查是否识别到新的COM端口。如果没有，请检查USB线连接并确保安装了正确的驱动程序。

  2. 配置串行终端

    • 在计算机上打开串行终端工具，例如PuTTY、SSCOM或类似工具。

    • 在串行终端工具中选择识别到的COM端口。

    • 将波特率设置为115200，停止位设置为1，数据位顺序设置为LSB，奇偶校验设置为NONE。

    • 点击连接按钮以建立与开发板的串行通信。

  3. 重置开发板

    • 在开发板上找到复位按钮。

    • 按下并释放复位按钮以重启开发板。

  4. 观察串行输出

    • 重置后，在串行终端工具中观察输出。程序将开始ADC采样。每隔0.5秒，串行终端将打印在AIN引脚测量到的电压值（范围为0-3.3V）。

    • 您可以更改连接到AIN引脚的电压，并观察串行打印的数据是否相应变化。这有助于确定ADC是否正常工作。

预期输出

  • 复位开发板后，串行终端将显示以下输出：

Ain = 0.687715
Ain = 0.707695

Ain = 0.739922

Overview

This document provides a guide to verify the functionality of the ADC (Analog-to-Digital Converter) peripheral using a development board (e.g., CB5000V210 Anchor demo board). It covers hardware connections, serial communication setup, and ADC sampling and voltage conversion.

Requirements

  • One of the following development boards:

    • CB5000V210 Anchor demo board

  • Software:

    • Serial terminal tool (e.g., PuTTY, SSCOM)

User interface

  • Serial Port Connections:

    • Connect GPIO00 to the serial receive line (RX).

    • Connect GPIO01 to the serial transmit line (TX).

  • AIN Port Connections:

    • Connect the AIN (Analog Input) on the demo board to the voltage source you want to measure (0-3.3V).

  • Ensure the connections are secure to avoid communication issues.

Building and running

  • This sample can be found under “Firmware\Examples\peripheral_adc” in the ChipsBank Connect SDK folder structure.

Testing

  1. Connect the Development Board

    • Prepare a suitable USB cable, ensuring both ends are compatible with your computer and demo board.

    • Insert one end of the USB cable into a USB port on your computer and the other end into the USB port on the demo board.

    • Open the Device Manager on your computer to check if a new COM port has been recognized. If not, verify the USB cable connection and ensure the correct drivers are installed.

  2. Configure the Serial Terminal

    • Open a serial terminal tool on your computer, such as PuTTY, SSCOM, or any similar tool.

    • In the serial terminal tool, select the recognized COM port.

    • Set the baud rate to 115200, stop bits to 1, data bits order to LSB, and parity to NONE.

    • Click the connect button to establish serial communication with the development board.

  3. Reset the Development Board

    • Locate the reset button on the demo board.

    • Press and release the reset button to restart the development board.

  4. Observe Serial Output

    • After resetting, observe the output in the serial terminal tool. The program will start ADC sampling. Every 0.5 seconds, the serial terminal should print the voltage value (in the range of 0 - 3.3V) measured at the AIN pin.

    • You can change the voltage connected to the AIN pin and observe if the serial-printed data changes accordingly. This helps to determine if the ADC is functioning correctly.
