APP-Gpio

概述

本文件提供了使用开发板验证GPIO外设功能的指南。它涵盖了硬件连接、串行通信设置以及通过UART控制LED和检测输入电平来测试GPIO操作的内容。

需求

  • 以下开发板之一：

    • CB5000V210 Anchor演示板

  • 软件：

    • 串行终端工具（例如PuTTY, SSCOM）

用户界面

  • 串口连接:

    • 将GPIO00连接到串行接收线（RX）。

    • 将GPIO01连接到串行发送线（TX）。

  • GPIO连接:

    • 将GPIO2和GPIO3设置为输出模式（带上拉），并分别连接到LED1和LED2。

    • 将GPIO5设置为边缘触发模式，用于测量外部中断（低电平触发，并在中断中翻转GPIO2的输出）。

    • 将GPIO4设置为输入模式，以检测高电平和低电平变化。

  • 确保所有连接牢固，以避免通信问题。

构建和运行

  • 该示例可以在ChipsBank Connect SDK文件夹结构中的”Firmware\Examples\peripheral_gpio”找到。

测试

  1. 连接开发板

    • 准备一个合适的USB线，确保两端与计算机和演示板兼容。

    • 将USB线的一端插入计算机的USB端口，另一端插入演示板的USB端口。

    • 打开计算机上的设备管理器，检查是否识别到新的COM端口。如果没有识别到，请检查USB线连接并确保安装了正确的驱动程序。

  2. 配置串行终端

    • 在计算机上打开一个串行终端工具，例如PuTTY, SSCOM或任何类似的工具。

    • 在串行终端工具中，选择识别到的COM端口。

    • 设置波特率为115200，停止位为1，数据位顺序为LSB，校验位为NONE。

    • 点击连接按钮以建立与开发板的串行通信。

  3. 重置开发板

    • 在演示板上找到复位按钮。

    • 按下并释放复位按钮以重启开发板。

  4. 执行测试操作

    • 复位后，在串行终端输出中观察。首先应看到启动消息“Power-on reset”。

    • 向GPIO4输入高电平和低电平，并检查LED2的状态。当GPIO4输入高电平时，LED2应点亮；当输入低电平时，LED2应熄灭。

    • 按下连接到GPIO5的按钮（以生成下降沿）。观察LED1的状态是否翻转。如果串口已连接，请检查串行终端是否打印进入中断的标志。

Overview

This document provides a guide to verify the functionality of the Gpio peripheral using a development board. It covers hardware connections, serial communication setup, and GPIO operation testing by controlling LEDs and detecting input levels via UART.

Requirements

  • One of the following development boards:

    • CB5000V210 Anchor demo board

  • Software:

    • Serial terminal tool (e.g., PuTTY, SSCOM)

User interface

  • Serial Port Connections:

    • Connect GPIO00 to the serial receive line (RX).

    • Connect GPIO01 to the serial transmit line (TX).

  • Ensure all connections are secure to avoid communication issues.

  • For GPIO related connections:

    • Set GPIO2 and GPIO3 as output mode (with pull-up) and connect them to LED1 and LED2 respectively.

    • Set GPIO5 as edge trigger mode for measuring external interrupt (triggered by low level, and invert the output of GPIO2 in the interrupt).

    • Set GPIO4 as input mode to detect high and low level changes.

Building and running

  • This sample can be found under “Firmware\Examples\peripheral_gpio” in the ChipsBank Connect SDK folder structure.

Testing

  1. Connect the Development Board

    • Prepare a suitable USB cable, ensuring both ends are compatible with your computer and demo board.

    • Insert one end of the USB cable into a USB port on your computer and the other end into the USB port on the demo board.

    • Open the Device Manager on your computer to check if a new COM port has been recognized. If not, verify the USB cable connection and ensure the correct drivers are installed.

  2. Configure the Serial Terminal

    • Open a serial terminal tool on your computer, such as PuTTY, SSCOM, or any similar tool.

    • In the serial terminal tool, select the recognized COM port.

    • Set the baud rate to 115200, stop bits to 1, data bits order to LSB, and parity to NONE.

    • Click the connect button to establish serial communication with the development board.

  3. Reset the Development Board

    • Locate the reset button on the demo board.

    • Press and release the reset button to restart the development board.

  4. Execute Test Operations

    • After resetting, observe the serial terminal output. You should first see the power-on message: “Power-on reset”.

    • Input high and low levels to GPIO4 and check the state of LED2. When GPIO4 inputs a high level, LED2 should turn on, and when it inputs a low level, LED2 should turn off.

    • Press the button connected to GPIO5 (to generate a falling edge). Observe that the state of LED1 flips. If the serial port is connected, check if the serial terminal prints the flag indicating entering the interrupt.
