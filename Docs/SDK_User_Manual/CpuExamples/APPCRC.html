APP-CRC

概述

本文件提供了使用开发板（例如CB5000V210 Anchor演示板）验证CRC外设功能的指南。它涵盖了硬件连接、串行通信设置以及通过轮询和中断模式进行CRC计算验证的内容。

需求

  • 以下开发板之一：

    • CB5000V210 Anchor演示板

  • 软件：

    • 串行终端工具（例如PuTTY, SSCOM）

用户界面

  • 串口连接:

    • 将GPIO00连接到串行接收线（RX）。

    • 将GPIO01连接到串行发送线（TX）。

  • 确保所有连接牢固，以避免通信问题。

构建和运行

  • 该示例可以在ChipsBank Connect SDK文件夹结构中的”Firmware\Examples\peripheral_crc”找到。

测试

  1. 连接开发板

    • 准备一个合适的USB线，确保两端与计算机和演示板兼容。

    • 将USB线的一端插入计算机的USB端口，另一端插入演示板的USB端口。

    • 打开计算机上的设备管理器，检查是否识别到新的COM端口。如果没有识别到，请检查USB线连接并确保安装了正确的驱动程序。

  2. 配置串行终端

    • 在计算机上打开一个串行终端工具，例如PuTTY, SSCOM或任何类似的工具。

    • 在串行终端工具中，选择识别到的COM端口。

    • 设置波特率为115200，停止位为1，数据位顺序为LSB，校验位为NONE。

    • 点击连接按钮以建立与开发板的串行通信。

  3. 重置开发板

    • 在演示板上找到复位按钮。

    • 按下并释放复位按钮以重启开发板。

  4. 观察串行输出

    • 复位后，在串行终端工具中观察输出。首先应看到启动消息“Power-on reset”。

    • 随后，将看到CRC8、CRC16和CRC32计算结果的打印，例如“CRC-8 Result (with polling): 0xA6”。

  5. 在线计算验证

    • 打开浏览器并访问[在线CRC计算器](https://www.crccalc.com/?crc=123456789&method=&datatype=ascii&outtype=hex)网站。

    • 在网站的输入框中，选择ASCII输入模式。

    • 对于CRC8验证，输入“Hello World!”，选择CRC8计算方法，并选择十六进制输出，然后记录结果。

    • 对于CRC16验证，输入“APP SDK CRC-16 Demo”，选择CRC16计算方法，并选择十六进制输出，然后记录结果。

    • 对于CRC32验证，输入“APP SDK CRC-32 Demo”，选择CRC32计算方法，并选择十六进制输出，然后记录结果。

  6. 结果比较

    • 将在线计算器获得的CRC8、CRC16和CRC32结果与串行终端工具中显示的结果进行比较。

    • 如果结果匹配，则CRC计算正确。如果不匹配，请检查数据设置、串行端口配置或代码中的CRC计算函数。

预期输出

  • 复位开发板后，串行终端将显示以下输出：

Power-on reset

Data to be performed for CRC8 calculations:  Hello World !
CRC-8 Result (with polling): 0xA6
CRC Result (with IRQ): 0xA6
CRC-8 with IRQ finished!

Data to be performed for CRC16 calculations:  APP SDK CRC-16 Demo
CRC-16 Result (with polling): 0x1BFC
CRC Result (with IRQ): 0x1BFC
CRC-16 with IRQ finished!

Data to be performed for CRC32 calculations:  APP SDK CRC-32 Demo
CRC-32 Result (with polling): 0x3D8BF5F3
CRC Result (with IRQ): 0x3D8BF5F3
CRC-32 with IRQ finished!

Overview

This document provides a guide to verify the functionality of the CRC peripheral using a development board (e.g., CB5000V210 Anchor demo board). It covers hardware connections, serial communication setup, and CRC calculation validation through polling and interrupt modes.

Requirements

  • One of the following development boards:

    • CB5000V210 Anchor demo board

  • Software:

    • Serial terminal tool (e.g., PuTTY, SSCOM)

User interface

  • Serial Port Connections:

    • Connect GPIO00 to the serial receive line (RX).

    • Connect GPIO01 to the serial transmit line (TX).

  • Ensure all connections are secure to avoid communication issues.

Building and running

  • This sample can be found under “Firmware\Examples\peripheral_crc” in the ChipsBank Connect SDK folder structure.

Testing

  1. Connect the Development Board

    • Prepare a suitable USB cable, ensuring both ends are compatible with your computer and demo board.

    • Insert one end of the USB cable into a USB port on your computer and the other end into the USB port on the demo board.

    • Open the Device Manager on your computer to check if a new COM port has been recognized. If not, verify the USB cable connection and ensure the correct drivers are installed.

  2. Configure the Serial Terminal

    • Open a serial terminal tool on your computer, such as PuTTY, SSCOM, or any similar tool.

    • In the serial terminal tool, select the recognized COM port.

    • Set the baud rate to 115200, stop bits to 1, data bits order to LSB, and parity to NONE.

    • Click the connect button to establish serial communication with the development board.

  3. Reset the Development Board

    • Locate the reset button on the demo board.

    • Press and release the reset button to restart the development board.

  4. Observe Serial Output

    • After resetting, observe the output in the serial terminal tool. You should first see the power-on message “Power-on reset”.

    • Subsequently, you will see the CRC8, CRC16, and CRC32 calculation results printed, such as “CRC-8 Result (with polling): 0xA6”.

  5. Online Calculation Verification

    • Open a web browser and visit the [Online CRC Calculator](https://www.crccalc.com/?crc=123456789&method=&datatype=ascii&outtype=hex) website.

    • In the website’s input box, select ASCII input mode.

    • For CRC8 verification, enter “Hello World!”, select the CRC8 calculation method, and choose hexadecimal output, then record the result.

    • For CRC16 verification, enter “APP SDK CRC-16 Demo”, select the CRC16 calculation method, choose hexadecimal output, and record the result.

    • For CRC32 verification, enter “APP SDK CRC-32 Demo”, select the CRC32 calculation method, choose hexadecimal output, and record the result.

  6. Result Comparison

    • Compare the CRC8, CRC16, and CRC32 results obtained from the online calculator with the results displayed in the serial terminal tool.

    • If the results match, the CRC calculations are correct. If they do not match, check the data settings, serial port configuration, or CRC calculation functions in the code.

Expected Output

  • The following output will be displayed in the serial terminal after resetting the development board:

CRC-8 Result (with polling): 0xA6
CRC Result (with IRQ): 0xA6
CRC-8 with IRQ finished!

CRC-16 Result (with polling): 0x1BFC
CRC Result (with IRQ): 0x1BFC
CRC-16 with IRQ finished!
