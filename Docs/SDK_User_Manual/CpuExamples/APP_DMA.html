APP-DMA

概述

本文件提供了使用开发板验证DMA外设功能的指南。它涵盖了硬件连接、串行通信设置以及通过比较传输前后数据来验证DMA传输的内容。

需求

  • 以下开发板之一：

    • CB5000V210 Anchor演示板

  • 软件：

    • 串行终端工具（例如PuTTY, SSCOM）

用户界面

  • 串口连接:

    • 将GPIO00连接到串行接收线（RX）。

    • 将GPIO01连接到串行发送线（TX）。

  • 确保所有连接牢固，以避免通信问题。

构建和运行

  • 该示例可以在ChipsBank Connect SDK文件夹结构中的”Firmware\Examples\peripheral_dma”找到。

测试

  1. 连接开发板

    • 准备一个合适的USB线，确保两端与计算机和演示板兼容。

    • 将USB线的一端插入计算机的USB端口，另一端插入演示板的USB端口。

    • 打开计算机上的设备管理器，检查是否识别到新的COM端口。如果没有识别到，请检查USB线连接并确保安装了正确的驱动程序。

  2. 配置串行终端

    • 在计算机上打开一个串行终端工具，例如PuTTY, SSCOM或任何类似的工具。

    • 在串行终端工具中，选择识别到的COM端口。

    • 设置波特率为115200，停止位为1，数据位顺序为LSB，校验位为NONE。

    • 点击连接按钮以建立与开发板的串行通信。

  3. 重置开发板

    • 在演示板上找到复位按钮。

    • 按下并释放复位按钮以重启开发板。

  4. 观察串行输出

    • 复位后，在串行终端工具中观察输出。首先应看到启动消息“Power-on reset”。

    • 随后，在启用DMA传输之前，将打印源地址和目标地址的数据。

    • 在DMA传输完成后，将打印传输前后目标地址的数据。观察目标地址的数据是否与源地址的数据匹配。

  5. 结果验证

    • 如果DMA传输完成后目标地址的数据与源地址的数据匹配，则表示DMA传输正确，DMA模块可以正常使用。

    • 如果数据不匹配，请进一步排查DMA配置和连接问题。

Overview

This document provides a guide to verify the functionality of the DMA peripheral using a development board. It covers hardware connections, serial communication setup, and DMA transfer validation by comparing data before and after the transfer.

Requirements

  • One of the following development boards:

    • CB5000V210 Anchor demo board

  • Software:

    • Serial terminal tool (e.g., PuTTY, SSCOM)

User interface

  • Serial Port Connections:

    • Connect GPIO00 to the serial receive line (RX).

    • Connect GPIO01 to the serial transmit line (TX).

  • Ensure all connections are secure to avoid communication issues.

Building and running

  • This sample can be found under “Firmware\Examples\peripheral_dma” in the ChipsBank Connect SDK folder structure.

Testing

  1. Connect the Development Board

    • Prepare a suitable USB cable, ensuring both ends are compatible with your computer and demo board.

    • Insert one end of the USB cable into a USB port on your computer and the other end into the USB port on the demo board.

    • Open the Device Manager on your computer to check if a new COM port has been recognized. If not, verify the USB cable connection and ensure the correct drivers are installed.

  2. Configure the Serial Terminal

    • Open a serial terminal tool on your computer, such as PuTTY, SSCOM, or any similar tool.

    • In the serial terminal tool, select the recognized COM port.

    • Set the baud rate to 115200, stop bits to 1, data bits order to LSB, and parity to NONE.

    • Click the connect button to establish serial communication with the development board.

  3. Reset the Development Board

    • Locate the reset button on the demo board.

    • Press and release the reset button to restart the development board.

  4. Observe Serial Output

    • After resetting the development board, observe the output in the serial terminal tool.

    • You should first see the power-on message: “Power-on reset”.

    • Subsequently, before enabling the DMA transfer, the data at the source address and the data at the destination address will be printed.

    • After the DMA transfer is completed, the data at the source address and the data at the destination address after the transfer will be printed. Observe whether the data at the destination address matches the data at the source address.

  5. Result Verification

    • If the data at the destination address matches the data at the source address after the DMA transfer, it indicates that the DMA transfer is correct and the DMA module can be used normally.

    • If the data does not match, further troubleshoot the DMA configuration and connection issues.
