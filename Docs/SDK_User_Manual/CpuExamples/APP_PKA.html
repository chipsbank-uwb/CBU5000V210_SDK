APP-PKA

概述

本文件提供了使用开发板验证PKA外设功能的指南。该示例涵盖了硬件连接、串行通信设置以及通过AES加密和解密操作来验证PKA功能的内容。结果验证通过比较输出与预期结果来完成。

需求

  • 以下开发板之一：

    • CB5000V210 Anchor演示板

  • 软件：

    • 串行终端工具（例如PuTTY, SSCOM）

用户界面

  • 串口连接:

    • 将GPIO00连接到串行接收线（RX）。

    • 将GPIO01连接到串行发送线（TX）。

  • 确保所有连接牢固，以避免通信问题。

构建和运行

  • 该示例可以在ChipsBank Connect SDK文件夹结构中的”Firmware\Examples\peripheral_pka”找到。

测试

  1. 连接开发板

    • 准备一个合适的USB线，确保两端与计算机和演示板兼容。

    • 将USB线的一端插入计算机的USB端口，另一端插入演示板的USB端口。

    • 打开计算机上的设备管理器，检查是否识别到新的COM端口。如果没有识别到，请检查USB线连接并确保安装了正确的驱动程序。

  2. 配置串行终端

    • 在计算机上打开一个串行终端工具，例如PuTTY, SSCOM或任何类似的工具。

    • 在串行终端工具中，选择识别到的COM端口。

    • 设置波特率为115200，停止位为1，数据位顺序为LSB，校验位为NONE。

    • 点击连接按钮以建立与开发板的串行通信。

  3. 重置开发板

    • 在演示板上找到复位按钮。

    • 按下并释放复位按钮以重启开发板。

  4. 执行测试操作

    • 复位后，在串行终端输出中观察。首先应看到启动消息“Power-on reset”。

    • 系统将使用配置的参数初始化串口。

    • 接下来，系统将执行与PKA模块相关的以下操作：

      • 调用 APP_PKA_EncryptRsa1024() 生成签名。

      • 调用 APP_PKA_VerifyRsa1024() 验证生成的签名。

      • 调用 APP_PKA_RsaSignAndVerifyPrecompute() 演示使用预计算值。

    • 观察并记录串行终端中打印的输入数组、签名结果、解密结果和验证结果。

  5. 结果验证

    • 检查串行终端是否打印“Verified success: 1”。如果显示此消息，表示签名验证成功，PKA模块正常工作。

    • 如果未显示该消息或显示其他错误消息，请进一步排查PKA配置、函数调用或串行通信设置。

Overview

This document provides a guide to verify the functionality of the PKA peripheral using AES encryption and decryption operations. The example covers hardware connections, serial communication setup, and result validation by comparing outputs with expected results.

Requirements

  • One of the following development boards:

    • CB5000V210 Anchor demo board

  • Software:

    • Serial terminal tool (e.g., PuTTY, SSCOM)

User interface

  • Serial Port Connections:

    • Connect GPIO00 to the serial receive line (RX).

    • Connect GPIO01 to the serial transmit line (TX).

  • Ensure all connections are secure to avoid communication issues.

Building and running

  • This sample can be found under “Firmware\Examples\peripheral_pka” in the ChipsBank Connect SDK folder structure.

Testing

  1. Connect the Development Board

    • Prepare a suitable USB cable, ensuring both ends are compatible with your computer and demo board.

    • Insert one end of the USB cable into a USB port on your computer and the other end into the USB port on the demo board.

    • Open the Device Manager on your computer to check if a new COM port has been recognized. If not, verify the USB cable connection and ensure the correct drivers are installed.

  2. Configure the Serial Terminal

    • Open a serial terminal tool on your computer, such as PuTTY, SSCOM, or any similar tool.

    • In the serial terminal tool, select the recognized COM port.

    • Set the baud rate to 115200, stop bits to 1, data bits order to LSB, and parity to NONE.

    • Click the connect button to establish serial communication with the development board.

  3. Reset the Development Board

    • Locate the reset button on the demo board.

    • Press and release the reset button to restart the development board.

  4. Execute Test Operations

    • After resetting, observe the output in the serial terminal tool. You should first see the power-on message: “Power-on reset”.

    • The system will then initialize the serial port with the configured parameters.

    • Next, the system will perform the following operations related to the PKA module:

      • Call APP_PKA_EncryptRsa1024() to generate a signature.

      • Call APP_PKA_VerifyRsa1024() to verify the generated signature.

      • Call APP_PKA_RsaSignAndVerifyPrecompute() to demonstrate the use of precomputed values.

    • Observe and record the input array, signature result, decryption result, and verification result printed in the serial terminal.

  5. Result Verification

    • Check if the serial terminal prints “Verified success: 1”. If this message is displayed, it indicates that the signature verification is successful and the PKA module is functioning properly.

    • If the message is not displayed or other error messages are shown, further troubleshoot the PKA configuration, function calls, or serial communication setup.
