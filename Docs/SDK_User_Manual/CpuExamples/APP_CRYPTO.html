APP-CRYPTO

概述

本文档提供了使用加密外设验证AES加密和解密操作（ECB、CMAC、CCM模式）的指南。示例涵盖了硬件连接、串行通信设置以及通过与在线工具比较输出结果来验证结果。

要求

  • 以下开发板之一：

    • CB5000V210 Anchor开发板

  • 软件：

    • 串行终端工具（例如PuTTY, SSCOM）

用户界面

  • 串口连接:

    • 将GPIO00连接到串行接收线（RX）。

    • 将GPIO01连接到串行发送线（TX）。

  • 确保所有连接牢固，以避免通信问题。

构建和运行

  • 该示例可以在ChipsBank Connect SDK文件夹结构中的”Firmware\Examples\peripheral_crypto”找到。

测试

  1. 连接开发板

    • 准备一个合适的USB线，确保两端与您的计算机和开发板兼容。

    • 将USB线的一端插入计算机的USB端口，另一端插入开发板的USB端口。

    • 打开计算机上的设备管理器，检查是否识别到新的COM端口。如果没有，请检查USB线连接并确保安装了正确的驱动程序。

  2. 配置串行终端

    • 在计算机上打开串行终端工具，例如PuTTY、SSCOM或类似工具。

    • 在串行终端工具中选择识别到的COM端口。

    • 将波特率设置为115200，停止位设置为1，数据位顺序设置为LSB，奇偶校验设置为NONE。

    • 点击连接按钮以建立与开发板的串行通信。

  3. 重置开发板

    • 在开发板上找到复位按钮。

    • 按下并释放复位按钮以重启开发板。

  4. 观察串行输出

    重置开发板后，在串行终端工具中观察输出。 您应该首先看到启动消息：”Power-on reset”。 随后，将打印以下模式的加密和解密结果：

    • AES-128 ECB加密和解密

    • AES-128 CMAC加密

    • AES-128 CCM加密和解密

    所有加密和解密操作使用以下固定数据：

    明文（输入数据）： 0x01, 0x23, 0x45, 0x67, 0x89, 0x10, 0x11, 0x12, 0x13, 0x14, 0x15, 0x16, 0x17, 0x18, 0x19, 0x20

    密钥： 0x00, 0x01, 0x02, 0x03, 0x04, 0x05, 0x06, 0x07, 0x08, 0x09, 0x0a, 0x0b, 0x0c, 0x0d, 0x0e, 0x0f

  5. 在线验证加密/解密结果

    用户可以通过访问在线AES加密/解密工具网站（例如Online AES Tool或其他类似工具）来验证加密和解密结果。

    在网站的输入框中输入以下数据：

    输入数据： 0x01, 0x23, 0x45, 0x67, 0x89, 0x10, 0x11, 0x12, 0x13, 0x14, 0x15, 0x16, 0x17, 0x18, 0x19, 0x20

    密钥： 0x00, 0x01, 0x02, 0x03, 0x04, 0x05, 0x06, 0x07, 0x08, 0x09, 0x0a, 0x0b, 0x0c, 0x0d, 0x0e, 0x0f

    选择适当的加密或解密模式（AES-128 ECB、AES-128 CMAC或AES-128 CCM），并将输出格式选择为十六进制（Hexadecimal）。 记录网站生成的结果。

  6. 结果比较

    • 将在线工具获得的结果与串行终端中显示的相应输出进行比较。

    • 如果结果匹配，则CRYPTO模块计算正确。

Overview

This document provides a guide to verify the functionality of the crypto peripheral using AES encryption and decryption operations (ECB, CMAC, CCM modes). The example covers hardware connections, serial communication setup, and result validation by comparing outputs with online tools.

Requirements

  • One of the following development boards:

    • CB5000V210 Anchor demo board

  • Software:

    • Serial terminal tool (e.g., PuTTY, SSCOM)

User interface

  • Serial Port Connections:

    • Connect GPIO00 to the serial receive line (RX).

    • Connect GPIO01 to the serial transmit line (TX).

  • Ensure all connections are secure to avoid communication issues.

Building and running

  • This sample can be found under “Firmware\Examples\peripheral_crypto” in the ChipsBank Connect SDK folder structure.

Testing

  1. Connect the Development Board

    • Prepare a suitable USB cable, ensuring both ends are compatible with your computer and demo board.

    • Insert one end of the USB cable into a USB port on your computer and the other end into the USB port on the demo board.

    • Open the Device Manager on your computer to check if a new COM port has been recognized. If not, verify the USB cable connection and ensure the correct drivers are installed.

  2. Configure the Serial Terminal

    • Open a serial terminal tool on your computer, such as PuTTY, SSCOM, or any similar tool.

    • In the serial terminal tool, select the recognized COM port.

    • Set the baud rate to 115200, stop bits to 1, data bits order to LSB, and parity to NONE.

    • Click the connect button to establish serial communication with the development board.

  3. Reset the Development Board

    • Locate the reset button on the demo board.

    • Press and release the reset button to restart the development board.

  4. Observe Serial Output

    After resetting the development board, observe the output in the serial terminal tool. You should first see the power-on message: “Power-on reset”. Subsequently, the encryption and decryption results for the following modes will be printed:

    • AES-128 ECB Encryption and Decryption

    • AES-128 CMAC Encryption

    • AES-128 CCM Encryption and Decryption

    All encryption and decryption operations use the following fixed data:

    Plaintext (Input Data): 0x01, 0x23, 0x45, 0x67, 0x89, 0x10, 0x11, 0x12, 0x13, 0x14, 0x15, 0x16, 0x17, 0x18, 0x19, 0x20

    Key: 0x00, 0x01, 0x02, 0x03, 0x04, 0x05, 0x06, 0x07, 0x08, 0x09, 0x0a, 0x0b, 0x0c, 0x0d, 0x0e, 0x0f

  5. Online Verification for Encryption/Decryption Results

    Users can verify the encryption and decryption results by visiting an online AES encryption/decryption tool website (e.g., Online AES Tool or similar tools).

    In the website’s input box, enter the following data:

    Input Data: 0x01, 0x23, 0x45, 0x67, 0x89, 0x10, 0x11, 0x12, 0x13, 0x14, 0x15, 0x16, 0x17, 0x18, 0x19, 0x20

    Key: 0x00, 0x01, 0x02, 0x03, 0x04, 0x05, 0x06, 0x07, 0x08, 0x09, 0x0a, 0x0b, 0x0c, 0x0d, 0x0e, 0x0f

    Select the appropriate encryption or decryption mode (AES-128 ECB, AES-128 CMAC, or AES-128 CCM) and choose the output format as hexadecimal (Hexadecimal). Record the result generated by the website.

  6. Result Comparison

    • Compare the results obtained from the online tools with the corresponding outputs displayed in the serial terminal.

    • If the results match, the CRYPTO module calculations are correct.
